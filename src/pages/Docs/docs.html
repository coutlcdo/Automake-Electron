Read the instructions carefully:

The algorithm right now will only work if you setup the environment correctly.

Instructions for the Environment Setup

You need to have both gcc and MinGW properly installed and setted at PATH.

Important! Inside your MinGW instalation folder, go to /bin and be sure the file mingw32-make is exactly as written here.

Instructions for the Watcher (main functionality)

First step:

Search for the folder where you keep your Allegro projects. For proper operation, I suggest that you create a folder to keep only the .c files and their corresponding assets.

Second step:

Search the folder where your Allegro library were downloaded. The name should contain the Allegro version.

Third step:

Now you should confirm if the path on text fields are correct and in the right order (the first text field should have Allegro projects folder path and the second should have Allegro library folder path)

If everything is correct you can Submit.

Once the paths have been submitted, you'll be redirected to the Watcher window. This means that Automake is already reading changes in the directory, you can now keep the app on background and work on your projects. Every time you save your project the app will automatically compile it into an executable.

Instructions for the Cleaner (secondary functionality)

Important! The call to the Cleaner will only work if at least the Allegro folder was submitted.

The Cleaner can be called clicking on the Options or simply hitting Ctrl+W (Command+W on Mac).

To check the source code go to Help and Github page.

Click Home or Ctrl+H (Command+H on Mac) to go back to the homepage.
