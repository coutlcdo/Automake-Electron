Read the instructions carefully:

Right now, the algorithm will only work if you setup the environment correctly.

Since MinGW is needed, Windows is recommended.

Instructions for the environment setup

You need to have MinGW properly installed and setted at PATH. Links: Allegro // Installer // Guide

Important! Inside your MinGW instalation folder, go to /bin and be sure to rename the file mingw32-make to make.

Remember to put the allegro-monolith dll inside your project folder.

Instructions to g++ compiling

Important! Changing compilers may cause unexpected errors, I suggest you that select the compiler before anything else and only once per runtime.

Some aditional evironment setup is needed before compiling with g++.

First step:

Enter the Allegro folder and copy all the files inside lib folder.

Second step:

Paste the files inside the lib folder of MinGW.

Third step:

Repeat first and second step but for include folder.

You'll need the allegro-monolith dll inside your project folder for this process too.

Instructions for the Watcher

First step:

Search for the folder where you keep your Allegro projects. To avoid errors, I suggest that you create a folder to keep only the .c files.

Second step:

Search the folder where your Allegro library were downloaded. The folder name should follow this template: allegro-(version)-mingw-(version).

Third step:

Now you should confirm if the path on text fields are correct and in the right order (the first text field should have Allegro projects folder path and the second should have Allegro library folder path)

If everything is correct you can Submit.

Once the paths have been submitted, you'll be redirected to the Watcher window. This means that Automake is already reading changes in the directory, you can now keep the app on background and work on your projects. Every time you save your project, the app will automatically compile it into an executable.

Instructions for the Cleaner

Important! Calling the Cleaner will only work if at least the Allegro folder was submitted.

The Cleaner will delete all .o and .exe files at your selected Allegro folder. It can be called selecting on the fourth option of File at the menu, or simply hitting Ctrl+W (Command+W on Mac).

Instructions for saving and opening compiled programs

Once you submit, a message box will appear, choose "Yes" if you want to save your path settings. Important! The settings are temporarily saved, so, they may disappear when you reboot your computer.

To load your last settings go to File and select the first option.

You can also open the last compiled program, or it's folder, directly from the app. But in order to that you'll have to first compile the program, making a modification on it or simply saving without any.

To load your last compiled program folder go to File and select the second option. If you want to open the last program itself select the third option or hit Ctrl+S (Command+S on Mac).

To check the source code go to Help and Learn More.

Click Home or hit Ctrl+H (Command+H on Mac) to go back to the homepage.
